## 2. 核心交互流程图 (Mermaid)

### 2.1 工单状态流转

```mermaid
stateDiagram-v2
    [*] --> 新建: 提交工单
    
    新建 --> 处理中: 分配处理人
    新建 --> 已关闭: 用户取消/驳回
    
    处理中 --> 待确认: 用户确认结果
    处理中 --> 处理中: 继续处理
    
    待确认 --> 已关闭: 用户接受
    待确认 --> 处理中: 需补充信息
    
    已关闭 --> [*]
```

### 2.2 SLA 时效监控流程

```mermaid
flowchart TD
    A[工单创建] --> B{SLA 1: 新建→处理中<br/>默认 2 小时}
    B -->|未超时| C[正常推进]
    B -->|超时未处理| D[⚠️ SLA 警告<br/>自动升级提醒]
    D --> E[主管介入]
    
    F[工单创建] --> G{SLA 2: 处理中→待确认<br/>默认 8 小时}
    G -->|未超时| C
    G -->|超时未确认| H[⚠️ SLA 警告<br/>催办提醒]
    H --> I[催办处理]
    
    C --> J{SLA 3: 待确认→已关闭<br/>默认 24 小时}
    J -->|未超时| K[正常关闭]
    J -->|超时未关闭| L[⚠️ SLA 警告<br/>延期关闭提醒]
    L --> M[加急处理]
```

## 5. 响应式布局建议

### 5.1 PC 端 (> 1024px)
- 工单列表采用双栏布局：左侧筛选 + 右侧列表
- 详情页采用三栏布局：左侧导航 + 中间内容 + 右侧关联信息

### 5.2 平板端 (768px - 1024px)
- 单栏卡片式布局
- 状态标签横向排列

### 5.3 移动端 (< 768px)
- 单栏垂直滚动
- 关键信息优先展示
- 状态流转记录折叠式设计
