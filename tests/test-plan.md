## 测试范围

### 核心业务流程
- 登录 → 创建工单 → 状态流转 → 关闭工单
- 状态流转: 新建 → 处理中 → 待确认 → 已关闭
- SLA 时效测试
- 权限管理

### 功能模块
- **工单管理**: 创建、查看、编辑、删除
- **用户管理**: 角色、权限
- **通知系统**: 分配通知、超时提醒
- **数据统计**: 工单报表

## 测试策略

### 1. 测试框架
```
/tests/
├── e2e/              # Playwright 端到端测试
├── api/              # API 接口测试
└── unit/             # 单元测试
```

### 2. 测试环境
- **前端**: React + Ant Design Pro
- **后端**: NestJS + PostgreSQL
- **测试工具**:
  - Playwright (E2E)
  - Jest (API)
  - Supertest (HTTP 请求)

### 3. 自动化策略
- **E2E 覆盖率**: 核心流程 80% 覆盖
- **API 覆盖率**: 所有接口 100% 覆盖
- **回归测试**: 每次版本更新自动执行
